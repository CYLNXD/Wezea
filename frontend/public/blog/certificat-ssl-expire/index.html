Qu'est-ce qu'un certificat SSL ?

Un certificat SSL (ou TLS, son successeur technique) est un fichier numérique qui authentifie l'identité de votre site web et chiffre les données échangées entre le navigateur du visiteur et votre serveur. C'est ce qui permet l'affichage du cadenas dans la barre d'adresse et l'utilisation du protocole https://.

Un certificat a une durée de validité limitée — généralement 90 jours pour Let's Encrypt et jusqu'à 1 an pour les certificats commerciaux. Passé cette date, il expire et les navigateurs le rejettent immédiatement.

WARN
Durée maximale depuis 2020
Apple, Google et Mozilla ont imposé une limite de 398 jours pour tous les certificats SSL publics. Depuis 2023, la tendance est à des certificats de 90 jours renouvelés automatiquement.

Conséquences d'un certificat SSL expiré

1. Vos visiteurs voient une page d'erreur bloquante

Chrome affiche "Votre connexion n'est pas privée" avec un avertissement rouge (NET::ERR_CERT_DATE_INVALID). Firefox affiche "Attention : risque probable de sécurité". La majorité des visiteurs ne contournent pas ces avertissements — ils ferment l'onglet.

2. Impact direct sur votre chiffre d'affaires

Les études montrent qu'un avertissement SSL fait fuir 85 % des visiteurs. Pour un site e-commerce ou SaaS, chaque heure d'expiration non détectée représente des ventes perdues. Les formulaires de contact ne fonctionnent plus, les paiements échouent, les inscriptions s'arrêtent.

3. Pénalité SEO

Google utilise HTTPS comme signal de classement depuis 2014. Un site en HTTP (ou avec SSL invalide) est pénalisé dans les résultats de recherche. De plus, Google Search Console remonte des erreurs de crawl sur les pages inaccessibles, ce qui dégrade l'indexation globale du site.

4. Rupture des intégrations techniques

Les API, webhooks, et outils tiers (CRM, analytics, paiement) refusent de communiquer avec un endpoint dont le certificat est invalide. Stripe, Brevo, HubSpot — tous vérifient la validité SSL avant chaque appel.

5. Atteinte à la réputation

Un SSL expiré signale un manque de sérieux dans la gestion technique. Pour une agence web ou une société qui gère des données clients, c'est particulièrement dommageable. Vos clients remarquent — et en parlent.

Comment détecter l'expiration d'un certificat SSL

Via le navigateur

Sur Chrome, cliquez sur le cadenas (ou l'icône d'avertissement) dans la barre d'adresse → La connexion est sécurisée → Le certificat est valide. Vous verrez la date d'expiration exacte.

Via la ligne de commande

openssl s_client -connect votre-domaine.com:443 -servername votre-domaine.com </dev/null 2>/dev/null \
  | openssl x509 -noout -dates

La sortie affiche notBefore (date d'émission) et notAfter (date d'expiration).

Comment renouveler son certificat SSL

Let's Encrypt (gratuit, le plus courant)

Let's Encrypt émet des certificats gratuits valables 90 jours. La commande de renouvellement dépend de votre configuration : 

# Avec Certbot (le client Let's Encrypt officiel)
sudo certbot renew

# Forcer le renouvellement même si le certificat est encore valide
sudo certbot renew --force-renewal

Après renouvellement, rechargez nginx ou Apache pour qu'il prenne en compte le nouveau certificat : 

sudo systemctl reload nginx
# ou
sudo systemctl reload apache2

Certificat commercial (payant)

Pour les certificats issus d'une autorité commerciale (DigiCert, Sectigo, GlobalSign), le processus passe par votre hébergeur ou revendeur. Vous recevez généralement un email de rappel 30 jours avant l'expiration — si vous avez configuré les alertes.

Hébergement managé (OVH, Cloudflare, etc.)

La plupart des hébergeurs gèrent le renouvellement automatiquement si vous utilisez leur service SSL intégré. Vérifiez dans votre panneau de contrôle que l'option "renouvellement automatique" est bien activée.

Automatiser le renouvellement : les bonnes pratiques

Configurer un cron job Certbot

Certbot installe généralement un timer systemd ou une entrée cron qui tente le renouvellement deux fois par jour. Vérifiez qu'il est actif : 

# Vérifier le timer systemd
systemctl status certbot.timer

# Ou vérifier le cron
crontab -l | grep certbot

Mettre en place des alertes préventives

Ne comptez pas uniquement sur le renouvellement automatique. Configurez des alertes email 30 et 7 jours avant l'expiration. Un outil de monitoring de domaine peut faire ça pour vous automatiquement.

INFO
Règle des 3 semaines
Renouvelez toujours votre certificat au moins 21 jours avant son expiration. Cela laisse du temps pour diagnostiquer un problème si le renouvellement automatique échoue silencieusement.

Tester régulièrement

Un renouvellement automatique peut échouer sans que personne ne s'en rende compte : problème de DNS, changement de configuration serveur, quota Let's Encrypt atteint. Incluez une vérification SSL dans votre routine de monitoring.

Expired SSL certificate: consequences, detection and renewal

Your SSL certificate has expired — or will expire in a few days. Result: a red warning page appears to your visitors, your bounce rate explodes, and Google downgrades your site. Here's everything you need to know to act fast.

What is an SSL certificate?

An SSL certificate (or TLS, its technical successor) is a digital file that authenticates your website's identity and encrypts data exchanged between the visitor's browser and your server. It's what displays the padlock in the address bar and enables the https:// protocol.

A certificate has a limited validity period — typically 90 days for Let's Encrypt and up to 1 year for commercial certificates. After that date, it expires and browsers reject it immediately.

WARN
Maximum validity since 2020
Apple, Google and Mozilla have imposed a 398-day limit for all public SSL certificates. Since 2023, the trend is toward 90-day certificates renewed automatically.

Consequences of an expired SSL certificate

1. Your visitors see a blocking error page

Chrome displays "Your connection is not private" with a red warning (NET::ERR_CERT_DATE_INVALID). Firefox shows "Warning: Potential Security Risk Ahead". Most visitors don't bypass these warnings — they close the tab.

2. Direct impact on your revenue

Studies show that an SSL warning scares away 85% of visitors. For an e-commerce or SaaS site, each hour of undetected expiration means lost sales. Contact forms stop working, payments fail, signups halt.

3. SEO penalty

Google has used HTTPS as a ranking signal since 2014. A site on HTTP (or with invalid SSL) is penalized in search results. Plus, Google Search Console reports crawl errors on inaccessible pages, degrading your site's overall indexation.

4. Break in technical integrations

APIs, webhooks, and third-party tools (CRM, analytics, payment) refuse to communicate with an endpoint with an invalid certificate. Stripe, Brevo, HubSpot — all verify SSL validity before every call.

5. Damage to reputation

An expired SSL signals carelessness in technical management. For a web agency or company handling customer data, this is particularly damaging. Your clients notice — and talk about it.

How to detect SSL expiration

Via the browser

On Chrome, click the padlock (or warning icon) in the address bar → Connection is secure → Certificate is valid. You'll see the exact expiration date.

Via the command line

openssl s_client -connect your-domain.com:443 -servername your-domain.com </dev/null 2>/dev/null \
  | openssl x509 -noout -dates

The output shows notBefore (issue date) and notAfter (expiration date).

Via automated tool

Monitoring tools like Wezea automatically verify your domains' SSL validity and alert you before expiration. No more catastrophic discoveries on a Monday morning.

How to renew your SSL certificate

Let's Encrypt (free, most common)

Let's Encrypt issues free certificates valid for 90 days. The renewal command depends on your setup: 

# With Certbot (the official Let's Encrypt client)
sudo certbot renew

# Force renewal even if certificate is still valid
sudo certbot renew --force-renewal

After renewal, reload nginx or Apache to pick up the new certificate: 

sudo systemctl reload nginx
# or
sudo systemctl reload apache2

Commercial certificate (paid)

For certificates from commercial authorities (DigiCert, Sectigo, GlobalSign), the process goes through your hosting provider or reseller. You typically receive a reminder email 30 days before expiration — if you've configured alerts.

Managed hosting (OVH, Cloudflare, etc.)

Most hosting providers handle renewal automatically if you use their integrated SSL service. Check your control panel that "automatic renewal" is enabled.

Automate renewal: best practices

Set up a Certbot cron job

Certbot typically installs a systemd timer or cron entry that attempts renewal twice daily. Verify it's active: 

# Check the systemd timer
systemctl status certbot.timer

# Or check the cron
crontab -l | grep certbot

Set up preventive alerts

Don't rely solely on automatic renewal. Configure email alerts 30 and 7 days before expiration. A domain monitoring tool can do this automatically for you.

INFO
The 3-week rule
Always renew your certificate at least 21 days before expiration. This gives you time to diagnose an issue if automatic renewal fails silently.

Test regularly

Automatic renewal can fail without anyone noticing: DNS issue, server config change, Let's Encrypt quota reached. Include SSL verification in your monitoring routine.
