Sécuriser WordPress : checklist complète 2026

WordPress fait tourner 43% du web mondial — ce qui en fait aussi la cible numéro un des attaques automatisées. Un WordPress non maintenu est compromis en moyenne en moins de 72 heures après exposition. Cette checklist couvre tous les niveaux : accès, mises à jour, configuration, email et monitoring.

1. Mises à jour — la base non négociable

Selon Sucuri, 52% des sites WordPress compromis en 2025 avaient au moins un composant non mis à jour au moment de l'attaque. Les pirates exploitent les CVE publiées dans les heures qui suivent leur annonce.

  • Mettre à jour WordPress core dès chaque release
  • Mettre à jour tous les plugins actifs ET inactifs
  • Mettre à jour tous les thèmes installés (même ceux non utilisés)
  • Activer les mises à jour automatiques pour WordPress core
  • Supprimer les plugins et thèmes inutilisés — ils restent des vecteurs d'attaque
WARN
Plugins abandonnés
Un plugin qui n'a pas été mis à jour depuis plus de 2 ans est probablement abandonné par son développeur. Les vulnérabilités ne seront jamais corrigées. Remplacez-le par une alternative active.

2. Contrôle des accès

Page de connexion

  • Changer l'URL de connexion (par défaut /wp-admin ou /wp-login.php — trop prévisible)
  • Activer l'authentification à deux facteurs (2FA) pour tous les administrateurs
  • Limiter les tentatives de connexion (plugin Limit Login Attempts Reloaded)
  • Ne jamais utiliser "admin" comme identifiant

Gestion des utilisateurs

  • Appliquer le principe du moindre privilège : un rédacteur n'a pas besoin du rôle Administrateur
  • Supprimer les comptes inactifs et les comptes de test
  • Utiliser des mots de passe forts (minimum 16 caractères, générés aléatoirement)
  • Désactiver l'enregistrement public si vous n'en avez pas besoin

Accès serveur

  • Utiliser une clé SSH à la place d'un mot de passe FTP
  • Désactiver FTP — utiliser SFTP uniquement
  • Restreindre l'accès SSH par IP si possible

3. Sécurité des fichiers

Permissions fichiers et dossiers

# Permissions recommandées
Fichiers WordPress : 644
Dossiers WordPress : 755
wp-config.php : 440 ou 400
.htaccess : 644

Protection wp-config.php

Ce fichier contient les credentials de votre base de données. Déplacez-le un niveau au-dessus du dossier public ou ajoutez dans .htaccess :

<files wp-config.php>
  order allow,deny
  deny from all
</files>

Désactiver l'édition de fichiers depuis le dashboard

Ajoutez dans wp-config.php :

define('DISALLOW_FILE_EDIT', true);
INFO
Désactiver l'exécution PHP dans uploads
Un vecteur d'attaque classique : uploader un fichier PHP malveillant déguisé en image. Ajoutez un .htaccess dans wp-content/uploads/ avec php_flag engine off pour empêcher l'exécution PHP dans ce dossier.

4. SSL et HTTPS

  • Certificat SSL valide et non expiré (vérifiez la date d'expiration)
  • Redirection automatique HTTP → HTTPS dans .htaccess ou wp-config.php
  • HSTS activé pour éviter les attaques downgrade
  • Zéro mixed content (images, scripts en HTTP sur une page HTTPS)
  • Support TLS 1.2 minimum, TLS 1.3 recommandé

5. Sécurité email du domaine

Votre site WordPress envoie probablement des emails (notifications, réinitialisations de mot de passe, newsletters). Si SPF, DKIM et DMARC ne sont pas configurés, ces emails arrivent en spam — et votre domaine peut être usurpé pour du phishing.

  • Enregistrement SPF couvrant votre serveur d'envoi WordPress
  • DKIM signé pour votre domaine (via votre hébergeur ou outil d'envoi)
  • DMARC configuré au minimum en p=none avec adresse RUA

6. Plugins de sécurité recommandés

  • Wordfence — firewall applicatif, scanner malware, blocage par IP. Version gratuite très complète.
  • WP Cerber — protection anti-brute-force et anti-spam robuste.
  • iThemes Security — durcissement WordPress, détection de changements de fichiers.
  • WP Activity Log — journal d'audit complet de toutes les actions sur le site.
  • UpdraftPlus — sauvegardes automatiques vers cloud (pas un plugin de sécurité mais essentiel).
WARN
N'installez pas plusieurs plugins de sécurité
Deux pare-feux applicatifs simultanés créent des conflits, ralentissent le site et donnent une fausse impression de double protection. Choisissez-en un et configurez-le bien.

7. Sauvegardes — votre dernier recours

Une bonne stratégie de sauvegarde WordPress suit la règle 3-2-1 : 3 copies, sur 2 supports différents, dont 1 hors site.

  • Sauvegarde automatique quotidienne de la base de données et des fichiers
  • Stockage hors serveur (S3, Google Drive, Dropbox)
  • Test de restauration au moins une fois par trimestre
  • Conserver 30 jours d'historique minimum

8. Monitoring continu

La sécurité n'est pas un état, c'est un processus. Un site peut être compromis silencieusement (injection de code dans un plugin, redirection cachée) sans symptôme visible pendant des semaines.

  • Scanner régulièrement les malwares (Wordfence, Sucuri)
  • Monitorer les changements de fichiers critiques (wp-config.php, functions.php)
  • Suivre l'état du certificat SSL et l'expiration du domaine
  • Vérifier régulièrement la réputation IP et la présence en liste noire
  • Activer les alertes Google Search Console pour les problèmes de sécurité

Securing WordPress: complete checklist 2026

WordPress powers 43% of the web — and is also the #1 target for automated attacks. An unpatched WordPress site is compromised on average within 72 hours of exposure. This checklist covers all levels: access, updates, configuration, email, and monitoring.

1. Updates — the non-negotiable foundation

According to Sucuri, 52% of WordPress sites compromised in 2025 had at least one outdated component at the time of attack. Hackers exploit CVEs published hours after announcement.

  • Update WordPress core with every release
  • Update all active AND inactive plugins
  • Update all installed themes (even unused ones)
  • Enable automatic updates for WordPress core
  • Delete unused plugins and themes — they remain attack vectors
WARN
Abandoned plugins
A plugin not updated in over 2 years is likely abandoned by its developer. Vulnerabilities will never be fixed. Replace it with an active alternative.

2. Access control

Login page

  • Change the login URL (default /wp-admin or /wp-login.php are too predictable)
  • Enable two-factor authentication (2FA) for all administrators
  • Limit login attempts (Limit Login Attempts Reloaded plugin)
  • Never use "admin" as a username

User management

  • Apply the principle of least privilege: editors don't need Administrator role
  • Delete inactive and test accounts
  • Use strong passwords (minimum 16 characters, randomly generated)
  • Disable public registration if you don't need it

Server access

  • Use SSH keys instead of FTP passwords
  • Disable FTP — use SFTP only
  • Restrict SSH access by IP if possible

3. File security

File and folder permissions

# Recommended permissions
WordPress files: 644
WordPress folders: 755
wp-config.php: 440 or 400
.htaccess: 644

Protect wp-config.php

This file contains your database credentials. Move it one level above the public folder or add to .htaccess:

<files wp-config.php>
  order allow,deny
  deny from all
</files>

Disable file editing from dashboard

Add to wp-config.php:

define('DISALLOW_FILE_EDIT', true);
INFO
Disable PHP execution in uploads
A classic attack vector: uploading a malicious PHP file disguised as an image. Add a .htaccess in wp-content/uploads/ with php_flag engine off to prevent PHP execution in that folder.

4. SSL and HTTPS

  • Valid, non-expired SSL certificate (check expiration date)
  • Automatic HTTP → HTTPS redirect in .htaccess or wp-config.php
  • HSTS enabled to prevent downgrade attacks
  • Zero mixed content (images and scripts over HTTP on HTTPS pages)
  • TLS 1.2 minimum, TLS 1.3 recommended

5. Domain email security

Your WordPress site likely sends emails (notifications, password resets, newsletters). Without SPF, DKIM, and DMARC configured, these emails land in spam — and your domain can be spoofed for phishing.

  • SPF record covering your WordPress sending server
  • DKIM signed for your domain (via your host or sending tool)
  • DMARC configured at minimum in p=none mode with RUA address

6. Recommended security plugins

  • Wordfence — Web application firewall, malware scanner, IP blocking. Free version is very complete.
  • WP Cerber — Robust brute-force and spam protection.
  • iThemes Security — WordPress hardening, file change detection.
  • WP Activity Log — Complete audit log of all site actions.
  • UpdraftPlus — Automatic backups to cloud (not security but essential).
WARN
Don't install multiple security plugins
Two application firewalls simultaneously create conflicts, slow down your site, and give false sense of double protection. Choose one and configure it well.

7. Backups — your last resort

Good WordPress backup strategy follows the 3-2-1 rule: 3 copies, on 2 different media, with 1 off-site.

  • Automatic daily backup of database and files
  • Off-server storage (S3, Google Drive, Dropbox)
  • Test restoration at least once per quarter
  • Keep minimum 30 days of history

8. Continuous monitoring

Security is not a state, it is a process. A site can be silently compromised (code injection in a plugin, hidden redirect) with no visible symptoms for weeks.

  • Regularly scan for malware (Wordfence, Sucuri)
  • Monitor critical file changes (wp-config.php, functions.php)
  • Track SSL certificate status and domain expiration
  • Regularly check IP reputation and blacklist status
  • Enable Google Search Console alerts for security issues
