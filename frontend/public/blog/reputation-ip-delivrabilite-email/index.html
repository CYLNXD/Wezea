Réputation IP et délivrabilité email : tout comprendre

Vous avez configuré SPF, DKIM et DMARC à la perfection — et pourtant vos emails arrivent systématiquement en spam. Le problème vient probablement de la réputation IP de votre serveur d'envoi. Voici comment ce système fonctionne, comment diagnostiquer le problème, et comment l'améliorer.

Comment fonctionne le scoring de réputation IP

Quand un email arrive sur le serveur de Gmail, Outlook ou Yahoo, ces serveurs ne se contentent pas de vérifier SPF et DKIM. Ils consultent également des bases de données de réputation pour évaluer l'historique de l'IP expéditrice.

Ce score de réputation est calculé en temps réel à partir de nombreux signaux. Une IP avec un bon historique d'envois légitimes, peu de rebonds et peu de signalements spam passera en boîte de réception. Une IP avec un historique douteux sera filtrée, quelle que soit la qualité du contenu de l'email.

INFO
Réputation IP ≠ réputation de domaine
Les deux coexistent. La réputation IP concerne l'adresse IP du serveur d'envoi. La réputation de domaine concerne le domaine dans l'adresse From. Un bon SPF/DKIM/DMARC améliore la réputation de domaine, mais ne répare pas une mauvaise réputation IP.

Les facteurs qui dégradent la réputation IP

Taux de rebond élevé

Les hard bounces (adresses inexistantes) signalent une liste de contacts mal qualifiée. Un taux de rebond supérieur à 2% est un signal négatif fort. Nettoyez régulièrement vos listes.

Taux de plaintes spam

Quand un destinataire clique "Signaler comme spam", les principaux FAI (Gmail, Outlook) le remontent à l'expéditeur via les boucles de rétroaction (FBL). Un taux de plainte supérieur à 0,1% commence à dégrader sérieusement la réputation.

Envoi vers des spamtraps

Les spamtraps sont des adresses email créées spécifiquement pour détecter les spammeurs. Elles ne s'inscrivent jamais volontairement à des newsletters. Si vous envoyez à ces adresses, c'est un signal fort que votre liste est de mauvaise qualité.

Volume d'envoi erratique

Passer de 10 emails/jour à 10 000 emails en une nuit est suspect. Les filtres interprètent ce type de comportement comme caractéristique d'une IP compromise.

Contenu marqué comme spam

Certains mots, structures ou patterns dans le contenu de l'email (liens raccourcis suspects, excès de majuscules, pièces jointes .zip) contribuent négativement — même si l'email est légitime.

Les principales blacklists

Une IP blacklistée voit ses emails refusés ou filtrés en spam par tous les serveurs qui consultent cette liste. Les principales listes utilisées par les serveurs de messagerie :

  • Spamhaus ZEN — la référence, combinaison de plusieurs listes Spamhaus (SBL, CBL, PBL)
  • Barracuda BRBL — très utilisée par les entreprises
  • SORBS — utilisée par de nombreux hébergeurs
  • SpamCop — basée sur les signalements des utilisateurs
  • MXToolbox Blacklist — agrège plus de 100 listes
WARN
Être blacklisté ne veut pas dire que vous spammez
Si vous utilisez un hébergeur cloud (AWS, OVH, Hetzner), votre IP peut avoir appartenu à un spammeur avant vous. Les IPs partagées sont souvent pré-blacklistées. Vérifiez dès le départ.

Vérifier sa réputation IP

Outils de vérification

  • MXToolbox Blacklist Check — vérifie votre IP sur plus de 100 listes
  • Sender Score (Validity) — score de 0 à 100 basé sur l'historique d'envoi
  • Google Postmaster Tools — réputation spécifique pour les envois vers Gmail
  • Microsoft SNDS — Smart Network Data Services, pour les envois vers Outlook/Hotmail
  • Wezea — vérifie la présence de votre domaine sur les principales blacklists

Identifier l'IP d'envoi concernée

Si vous utilisez un outil d'envoi comme Brevo, Mailchimp ou SendGrid, l'IP n'est pas la vôtre — c'est celle de leur serveur. En cas de problème, contactez leur support ou passez sur une IP dédiée.

Améliorer sa délivrabilité

Court terme — sortir d'une blacklist

La plupart des blacklists proposent un formulaire de délistage. Pour Spamhaus, le processus est automatique si la cause du listing est résolue. Pour Barracuda ou SORBS, vous soumettez une demande avec explication.

Important : résolvez d'abord le problème à l'origine du listing avant de demander le délistage. Sinon vous serez re-listé rapidement.

Moyen terme — améliorer la qualité des envois

  • Implémenter le double opt-in pour toutes les nouvelles inscriptions
  • Nettoyer les listes : supprimer les hard bounces, les inactifs depuis 12+ mois
  • Gérer les désabonnements immédiatement (obligation légale et signal de réputation)
  • Envoyer uniquement à des contacts qui ont explicitement consenti

Long terme — surveillance continue

  • Monitorer le Sender Score mensuellement
  • Configurer les FBL (Feedback Loop) auprès des principaux FAI
  • Analyser les rapports DMARC pour détecter les usurpations

IP partagée vs IP dédiée

Sur une IP partagée (cas de la plupart des outils d'envoi grand public), votre réputation dépend aussi du comportement des autres utilisateurs de la même IP. Sur une IP dédiée, vous êtes seul maître de votre réputation — mais vous devez la construire de zéro.

OK
IP dédiée : quand la choisir ?
Une IP dédiée n'est pertinente qu'à partir de ~50 000 emails/mois. En dessous, une IP partagée de bonne réputation (Brevo, Mailchimp, Sendinblue) est préférable car le volume d'envoi régulier maintient naturellement la réputation.

IP reputation and email deliverability: the full picture

You've configured SPF, DKIM, and DMARC perfectly — yet your emails consistently end up in spam. The problem likely comes from your sending server's IP reputation. Here's how this system works, how to diagnose the issue, and how to improve it.

How IP reputation scoring works

When an email arrives at Gmail, Outlook, or Yahoo servers, these services don't just check SPF and DKIM. They also consult reputation databases to evaluate the sender's IP history.

This reputation score is calculated in real-time from many signals. An IP with a good history of legitimate sends, few bounces, and few spam reports will reach the inbox. An IP with a questionable history will be filtered, regardless of email content quality.

INFO
IP reputation ≠ domain reputation
Both coexist. IP reputation concerns the sending server's IP address. Domain reputation concerns the domain in the From address. Good SPF/DKIM/DMARC improves domain reputation, but doesn't fix poor IP reputation.

Factors that degrade IP reputation

High bounce rate

Hard bounces (non-existent addresses) signal a poorly qualified contact list. A bounce rate over 2% is a strong negative signal. Regularly clean your lists.

High spam complaint rate

When a recipient clicks "Report as spam," major ISPs (Gmail, Outlook) report it via feedback loops (FBL). A complaint rate over 0.1% seriously degrades reputation.

Sending to spamtraps

Spamtraps are email addresses created specifically to catch spammers. They never voluntarily sign up for newsletters. Sending to these addresses signals poor list quality.

Erratic sending volume

Going from 10 emails/day to 10,000 emails overnight is suspicious. Filters interpret this behavior as characteristic of a compromised IP.

Content marked as spam

Certain words, structures, or patterns in email content (suspicious shortened links, excessive capitals, .zip attachments) contribute negatively — even if the email is legitimate.

Major blacklists

A blacklisted IP sees its emails refused or filtered as spam by all servers consulting that list. The main lists used by mail servers:

  • Spamhaus ZEN — the gold standard, combines multiple Spamhaus lists (SBL, CBL, PBL)
  • Barracuda BRBL — widely used by enterprises
  • SORBS — used by many hosting providers
  • SpamCop — based on user reports
  • MXToolbox Blacklist — aggregates 100+ lists
WARN
Being blacklisted doesn't mean you spam
If you use a cloud host (AWS, OVH, Hetzner), your IP may have belonged to a spammer before you. Shared IPs are often pre-blacklisted. Check from day one.

Check your IP reputation

Verification tools

  • MXToolbox Blacklist Check — checks your IP against 100+ lists
  • Sender Score (Validity) — 0-100 score based on sending history
  • Google Postmaster Tools — reputation specific to Gmail sends
  • Microsoft SNDS — Smart Network Data Services, for Outlook/Hotmail sends
  • Wezea — checks if your domain appears on major blacklists

Identify your sending IP

If you use a sending tool like Brevo, Mailchimp, or SendGrid, the IP isn't yours — it's theirs. If you have issues, contact their support or switch to a dedicated IP.

Improve your deliverability

Short term — get off a blacklist

Most blacklists offer a delisting form. For Spamhaus, the process is automatic if the listing cause is resolved. For Barracuda or SORBS, you submit a request with explanation.

Important: fix the problem causing the listing first before requesting delisting. Otherwise you'll be relisted quickly.

Medium term — improve send quality

  • Implement double opt-in for all new signups
  • Clean lists: remove hard bounces, inactive for 12+ months
  • Handle unsubscribes immediately (legal requirement and reputation signal)
  • Send only to contacts who explicitly consented

Long term — continuous monitoring

  • Monitor Sender Score monthly
  • Set up FBLs (Feedback Loops) with major ISPs
  • Analyze DMARC reports to detect spoofing

Shared vs dedicated IP

On a shared IP (most mainstream sending tools), your reputation also depends on other users' behavior. On a dedicated IP, you control your reputation — but must build it from scratch.

OK
Dedicated IP: when to choose it?
A dedicated IP only makes sense at ~50,000+ emails/month. Below that, a good-reputation shared IP (Brevo, Mailchimp, Sendinblue) is better since regular send volume naturally maintains reputation.
